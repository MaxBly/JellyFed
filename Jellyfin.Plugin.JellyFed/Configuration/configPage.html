JellyFed

Federation native d'instances Jellyfin

JellyFed connecte plusieurs serveurs Jellyfin entre eux. Depuis un seul client, tu accèdes aux bibliothèques de toutes les instances fédérées — avec artwork, métadonnées, pistes audio/sous-titres et transcodage HLS — de façon transparente pour tous les clients officiels.

Concept

[Client Jellyfin]
       |
       v
[Instance A  <---- JellyFed ---->  Instance B]
       |                                 |
  Bibliothèque A                   Bibliothèque B
  (locale)                         (partagée via .strm + NFO)

L'instance A installe JellyFed et se connecte à l'instance B. Le plugin synchronise le catalogue de B dans A sous forme de .strm + .nfo. Les clients voient les médias de B comme s'ils étaient locaux.

Setup rapide

  1. Onglet Settings : copie ton Federation Token (auto-généré) et partage-le avec tes peers — ils en auront besoin pour ajouter ta machine.
  2. Remplis Instance Name, Self URL (URL accessible depuis tes peers) et les racines Movies / Series / Anime. Optionnel : Jellyfin API Key pour du transcodage HLS natif.
  3. Onglet Peers → Add peer. Renseigne le nom, l'URL et le token partagés par le peer distant. L'auto-registration prévient l'autre côté automatiquement.
  4. Ajoute des bibliothèques Jellyfin pointant vers {MoviesRoot}, {SeriesRoot} et {AnimeRoot} (types Films / Séries selon ton usage). Les .strm sont rangés par peer dans {Racine}/{NomDuPeer}/.

Danger Zone

Resetting the network generates a new federation token, removes all peers and deletes all synced .strm files locally. Remote instances that had you as a peer will receive 401 errors and clean up automatically on their next sync. Use this if your token leaked or too many stale peers accumulated.
